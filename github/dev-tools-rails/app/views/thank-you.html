  1. Part One // The DOM: Elements & Styles

    1. Give Chrome access to your local folder for the application to enable autosaving.
    2. Put the form fields in order based on the quantity of each item being ordered.
    3. Let's remove the flip flops node from the order form, since nobody wants to wear flip flops in the winter time.
    4. Make sure users know they are about to click the submit button by adding a hover state that changes the background color, the cursor to a hand (cursor: pointer) and darkens the border.
    5. Create a new CSS rule for a selected element
    6. Left-align the text that specifies the color choices for hats and coats being ordered.

  2. Part Two // Debugging JavaScript

    1. It seems like nothing happens when we click the submit button. Let's investigate why.
    2. We've now verified that our submit function is being called, but it still seems like the order is not going through. Check the console for errors.
    3. Investigate the stack trace for the error. Using the sources tab, let's set a breakpoint where the error is occurring and take a look around.
    4. What is data type of the value we are trying to push to? Let's correct it our .push() method works.

  3. Part Three // Inspecting Network Requests

    1. Now that we've fixed our data type so our order form gets sent, let's navigate to the network tab and watch for our request to be sent.
    2. We are hoping for a Status Code of 200, OK. Unfortunately, this isn't what we get. Click on the request and review the data that was sent, as well as the response we received.
    3. Once we understand what caused the error, let's adjust our request body so we can get that 200 OK status.
